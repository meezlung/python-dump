Pepe the Pusher is always pushing something. You probably don't want to know all the things he's pushing. But you should know that Pepe the Pusher is a hardworking individual who has provided for all of his families.

Today, Pepe is pushing a box containing some substance. (Don't ask what the substance is.) He is tasked with pushing this box into one of the designated areas in the warehouse. However, it has just rained, and some parts of the warehouse floor has become slippery! This makes the task trickier because if the box or Pepe steps on a wet cell, they will slide until they hit until they land on a non-wet cell or hit something.

Task Details

We can represent the company warehouse as an r×cr×c grid. Each cell of the grid contains a character representing the contents of that cell and is one of the following:

  • A # indicates that the cell contains an obstacle.
  • A . indicates that the cell is empty and isn't wet.
  • A / indicates that the cell is empty and is wet.
  • A O indicates that the cell is empty, isn't wet, and is one of the designated areas that the box can be put into.
  • A Ø indicates that the cell is empty, is wet, and is one of the designated areas that the box can be put into.

We number the rows 00 to r−1r−1 from top to bottom and columns 00 to c−1c−1 from left to right, and denote by (i,j)(i,j) the cell at row ii and column jj. We represent the four cardinal directions as one of the following characters: U for up, D for down, L for left, and R for right.

This task has two parts. In the first part, you are given a sequence of actions to be performed by Pepe, and your goal is to simulate them. In the second part, your goal is to find a sequence of actions that solves the pushing puzzle! (You get points for each part.)

Your task is to implement a class called Pusher. This class should be instantiable by passing three arguments:

  • First, an r×cr×c grid (that is, an instance of Sequence[str]) representing the warehouse floor.
  • Second, a pair (iPepe,jPepe)(iPepe,jPepe) (of type tuple[int, int]) representing Pepe's initial location.
  • Third, a pair (ibox,jbox)(ibox,jbox) (of type tuple[int, int]) representing the initial location of the box.

In other words, the constructor must have the following signature:

Copy
    def __init__(self, grid: Sequence[str], pepe: tuple[int, int], box: tuple[int, int]) -> None:

The class must have the following methods (assuming p: Pusher):

  • p.move(dir: str) -> bool. This accepts a direction dir which must be one of the four cardinal directions, and means that Pepe steps toward that direction. It should return True if something actually happened—that is, the ending location is different from the starting location.
  • p.push(dir: str) -> bool. This accepts a direction dir which must be one of the four cardinal directions, and means that Pepe pushes the box in that direction, assuming the box is adjacent to Pepe in that direction; otherwise, this does nothing. It should return True if something actually happened—that is, the box's ending location is different from its starting location. Either way, Pepe's location will remain unchanged.
  • p.is_solved() -> bool. This returns True if the puzzle is solved, that is, the box is on top of a designated location, and False otherwise.
  • p.find_solution() -> list[tuple[str, str]] | None. This must return the shortest sequence of moves that solves the puzzle. Each element of the list must be (action, dir) where action is either move or push and dir is one of the four cardinal directions. If the puzzle is unsolvable, the method must return None. If there are multiple possible shortest answers, any one will be accepted.

Partial points will be given if you're able to implement all but the last method.

Note that when Pepe or the box moves toward a direction, they slide on wet floor, and they keep sliding until they reach a non-wet floor, hit the wall of the warehouse, or hit an obstacle. Note that Pepe also stops sliding if he hits the box.
